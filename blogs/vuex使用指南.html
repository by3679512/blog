    vuex使用指南

    舟羽 2023/6/10 vue

    # vuex使用指南

    # 1. vuex是什么

    vuex是一个专为vue.js应用程序开发的状态管理模式，它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

    # 2. 为什么使用vuex

    在大型单页应用中，组件之间的状态管理是一个很麻烦的事情，如果不加以限制，状态管理会变得非常混乱，难以维护。vuex就是为了解决这个问题而诞生的。

    # 3. vuex的核心概念

    # 3.1 state

    state是vuex中存储数据的地方，可以把state看作是一个全局变量，所有组件都可以访问到state中的数据。

    # 3.2 getters

    getters可以理解为vuex中的计算属性，getters的值会根据它的依赖被缓存起来，只有当它的依赖值发生了改变才会被重新计算。

    # 3.3 mutations

    mutations是vuex中唯一修改state的方式，每个mutation都有一个字符串的事件类型和一个回调函数，回调函数就是我们实际进行状态更改的地方，它接受state作为第一个参数。

    # 3.4 actions

    actions可以理解为vuex中的方法，它可以包含任意异步操作。

    # 3.5 modules

    modules可以理解为vuex中的子模块，它可以使得我们的store分割成多个模块，每个模块都有自己的state、getters、mutations、actions。

    # 4. vuex的使用

    # 4.1 安装vuex

    npm install vuex --save

    # 4.2 创建store

    import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    getCount: state => state.count
  },
  mutations: {
    increment (state) {
      state.count++
    }
  },
  actions: {
    increment ({ commit }) {
      commit('increment')
    }
  }
})

export default store

    # 4.3 在main.js中引入store

    import Vue from 'vue'
import App from './App.vue'
import store from './store'

new Vue({
  store,
  render: h => h(App)
}).$mount('#app')

    # 4.4 在组件中使用vuex

    <template>
  <div>
    <p>{{ count }}</p>
    <button @click="increment">increment</button>
  </div>
</template>

<script>
import { mapGetters, mapActions } from 'vuex'

export default {
  computed: {
    ...mapGetters(['getCount'])
  },
  methods: {
    ...mapActions(['increment'])
  }
}
</script>

    # 5. vuex的使用场景

    # 5.1 多个视图依赖于同一状态

    当多个视图依赖于同一状态时，来自不同视图的行为需要变更同一状态时，就需要使用vuex来管理状态。

    # 5.2 来自不同视图的行为需要变更同一状态

    当来自不同视图的行为需要变更同一状态时，就需要使用vuex来管理状态。

    # 5.3 某个状态需要在任何地方都可以被访问

    当某个状态需要在任何地方都可以被访问时，就需要使用vuex来管理状态。

    # 6. vuex的使用原则

    # 6.1 单一状态树

    vuex使用单一状态树，即每个应用将仅仅包含一个store实例，但是单一状态树和模块化并不冲突，vuex允许我们将store分割成模块，每个模块都有自己的state、getters、mutations、actions。

    # 6.2 状态是只读的

    vuex中的状态是只读的，唯一改变状态的方式就是显式的提交mutation，这样使得我们可以方便的跟踪每一个状态的变化。

    # 6.3 使用mutation进行状态更改

    vuex中的mutation类似于事件，每个mutation都有一个字符串的事件类型和一个回调函数，回调函数就是我们实际进行状态更改的地方，它接受state作为第一个参数。

    # 6.4 异步操作放在action中

    vuex中的action类似于mutation，但是它可以包含任意异步操作，action中的回调函数接受一个context对象，它包含了state、getters、commit、dispatch等属性和方法。

    # 6.5 组件中的数据应该来自vuex

    vuex中的state可以被任意组件访问，所以组件中的数据应该来自vuex，而不是直接在组件中定义。

    # 6.6 组件中的行为应该通过action来提交mutation

    vuex中的mutation是同步的，所以组件中的行为应该通过action来提交mutation。

    # 7. vuex的使用技巧

    # 7.1 使用mapState、mapGetters、mapMutations、mapActions

    vuex提供了mapState、mapGetters、mapMutations、mapActions这四个辅助函数，可以帮助我们在组件中更方便的使用vuex。

    <template>
  <div>
    <p>{{ count }}</p>
    <button @click="increment">increment</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapMutations, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState(['count']),
    ...mapGetters(['getCount'])
  },
  methods: {
    ...mapMutations(['increment']),
    ...mapActions(['increment'])
  }
}
</script>

    # 7.2 使用辅助函数

    vuex提供了辅助函数，可以帮助我们更方便的使用vuex。

    import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    getCount: state => state.count
  },
  mutations: {
    increment (state) {
      state.count++
    }
  },
  actions: {
    increment ({ commit }) {
      commit('increment')
    }
  }
})

export default store

    <template>
  <div>
    <p>{{ count }}</p>
    <button @click="increment">increment</button>
  </div>
</template>

<script>

export default {
  computed: {
    count () {
      return this.$store.state.count
    }
  },
  methods: {
    increment () {
      this.$store.commit('increment')
    }
  }
}
</script>
